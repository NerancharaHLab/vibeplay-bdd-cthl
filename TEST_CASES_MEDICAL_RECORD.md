# Test Cases: เวชระเบียน (Medical Record)

เอกสารฉบับนี้รวบรวมเทสเคสสำหรับโมดูลเวชระเบียน โดยเน้นที่ UI Test และ Functional Test

## 1. การสร้างประวัติใหม่ (Create New Patient)
- **MR-001 (UI)**: ตรวจสอบความครบถ้วนขององค์ประกอบในหน้าสร้างประวัติ (ฟิลด์ข้อมูล, ปุ่มบันทึก, ปุ่มยกเลิก)
- **MR-002 (UI)**: ตรวจสอบการแสดงผล Error Message เมื่อไม่ได้กรอกฟิลด์ที่จำเป็น (Mandatory Fields)
- **MR-003 (Functional)**: บันทึกข้อมูลผู้ป่วยใหม่สำเร็จ (Happy Path)
- **MR-004 (Functional)**: ตรวจสอบการแจ้งเตือนเมื่อกรอกเลขบัตรประชาชนซ้ำในระบบ
- **MR-005 (Functional)**: ตรวจสอบการทำงานของปุ่ม "อ่านบัตรประชาชน" (Smart Card Integration - ถ้ามีเครื่องทดสอบ)

## 2. การค้นหาผู้ป่วย (Patient Search)
- **MR-006 (Functional)**: ค้นหาด้วยเลข HN (ความแม่นยำ 100%)
- **MR-007 (Functional)**: ค้นหาด้วยชื่อ-นามสกุล (รองรับ Partial Search)
- **MR-008 (Functional)**: ค้นหาด้วยเลขบัตรประชาชน (ID Card / Passport)
- **MR-009 (UI)**: ตรวจสอบการแสดงผลตารางผลลัพธ์ (Pagination, Sorting)
- **MR-013 (UI)**: ตรวจสอบความครบถ้วนของฟิลด์ค้นหา (HN, ชื่อ-นามสกุล, บัตรประชาชน, เบอร์โทรศัพท์, VN, AN, วอร์ด, แพทย์, รหัสไปรษณีย์)
- **MR-014 (Functional)**: ค้นหาด้วยเลข HN และตรวจสอบความถูกต้องของข้อมูลในตาราง
- **MR-015 (Functional)**: ทดสอบปุ่ม "Clear" เพื่อล้างค่าในทุกฟิลด์การค้นหา

## 3. การเปิด Visit ผู้ป่วยนัด (Advance Visits)
- **AV-001 (UI)**: ตรวจสอบองค์ประกอบหน้าจอ "เปิด Visit ผู้ป่วยนัด" (หัวข้อหน้า, ปุ่มเพิ่มนัดหมาย)
- **AV-002 (UI)**: ตรวจสอบฟิลด์ตัวกรอง (วันที่นัดหมาย, คลินิก/หน่วยบริการ, แพทย์)
- **AV-003 (Functional)**: ทดสอบการกรองข้อมูลนัดหมายตามวันที่

## 4. การแก้ไขข้อมูล (Edit Profile)
- **MR-010 (Functional)**: แก้ไขข้อมูลที่อยู่และเบอร์โทรศัพท์ติดต่อ
- **MR-011 (Functional)**: แก้ไขผู้ติดต่อฉุกเฉิน (Emergency Contact)
- **MR-012 (UI/Functional)**: ตรวจสอบประวัติการแก้ไขข้อมูล (Audit Trail - ถ้ามีฟังก์ชันนี้)
