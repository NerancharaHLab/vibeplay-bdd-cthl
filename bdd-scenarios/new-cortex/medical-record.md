# BDD Scenarios: Medical Record / สร้างประวัติและแก้ไขข้อมูลผู้ป่วย (New Cortex)

พื้นที่สำหรับทีม QA ในการดูตัวอย่างเคสทดสอบที่แมปตามไฟล์ `TEST_CASES_MEDICAL_RECORD.md` อย่างเป็นระบบ

---

## 1. เคสทดสอบที่ผ่านการเขียนสคริปต์อัตโนมัติแล้ว (Automated Scenarios)

### 🚩 MR-003 (Functional): บันทึกข้อมูลผู้ป่วยใหม่สำเร็จ (Happy Path)
*คำอธิบาย:* ตรวจสอบความถูกต้องของการกรอกข้อมูลพื้นฐานผู้ป่วยใหม่ร่วมกับเลขบัตรประชาชน 13 หลักที่สุ่มสร้างขึ้น และระบบสามารถกรอกข้อมูลได้ถูกต้องสมบูรณ์

* **Given** ทีมงานล็อกอินเข้าสู่ระบบ Cortex Cloud ด้วยสิทธิ์ Super User
* **When** นำทางเข้าสู่ระบบโมดูลเวชระเบียน (Medical Record Application)
* **And** คลิกที่ปุ่ม "สร้างประวัติผู้ป่วยใหม่" (Create New Patient)
* **And** กรอกข้อมูล ชื่อจริงเป็น "TestName", นามสกุลเป็น "TestLastName" พร้อมรันโปรแกรมสร้างเลขบัตรประชาชน 13 หลักแบบสุ่ม
* **Then** ฟิลด์ข้อมูลในหน้าจอป้อนข้อมูลจะต้องแสดงค่า ชื่อจริง นามสกุล และเลขบัตรประชาชน 13 หลักอย่างถูกต้องครบถ้วน

---

## 2. ตัวอย่าง BDD สำหรับเคสทดสอบอื่นในแผนงาน (QA Backlog & Manual Scenarios)

> [!NOTE]
> ด้านล่างนี้คือการแปลงเคสทดสอบในไฟล์เวชระเบียนที่เหลือทั้งหมดให้กลายเป็นรูปแบบ BDD (Given-When-Then) เพื่อให้ทีม QA ใช้เป็นตัวอย่างในการเขียนและวางแผนเคสทดสอบในอนาคต

### 🚩 MR-001 (UI): ตรวจสอบความครบถ้วนขององค์ประกอบในหน้าสร้างประวัติ
*คำอธิบาย:* ตรวจสอบการแสดงผลของหน้าจอป้อนข้อมูลผู้ป่วยใหม่ว่ามีปุ่มและช่องกรอกข้อมูลสำคัญแสดงขึ้นครบถ้วนหรือไม่

* **Given** ผู้ใช้ล็อกอินเข้าระบบและเข้าสู่หน้าจอสร้างประวัติผู้ป่วยใหม่ (Create New Patient Page)
* **Then** ช่องกรอกข้อมูล ชื่อจริง (First Name), นามสกุล (Last Name) และ เลขบัตรประชาชน (ID Card Input) จะต้องแสดงผลขึ้นมา
* **And** ปุ่ม "บันทึกข้อมูล" (Save Button) และปุ่ม "ยกเลิก" (Cancel Button) จะต้องแสดงผลขึ้นมาและสามารถใช้งานได้

### 🚩 MR-002 (UI): ตรวจสอบการแสดงผล Error Message เมื่อไม่ได้กรอกฟิลด์ที่จำเป็น (Mandatory Fields)
*คำอธิบาย:* ตรวจสอบระบบดักจับการกรอกข้อมูล (Validation) โดยปล่อยช่องเว้นว่างแล้วกดบันทึก

* **Given** ผู้ใช้ล็อกอินเข้าระบบและเข้าสู่หน้าจอสร้างประวัติผู้ป่วยใหม่ (Create New Patient Page)
* **When** เว้นช่องกรอกข้อมูลที่จำเป็นทั้งหมดให้ว่างไว้ (เช่น ชื่อจริง นามสกุล หรือ เลขบัตรประชาชน)
* **And** ผู้ใช้พยายามคลิกปุ่ม "บันทึกข้อมูล"
* **Then** ระบบจะต้องแสดงข้อความเตือนผิดพลาด (Error Message/Validation Alert) สีแดงแจ้งเตือนให้กรอกข้อมูล
* **And** ระบบจะต้องไม่อนุญาตให้ทำการบันทึกข้อมูลสำเร็จ

### 🚩 MR-004 (Functional): ตรวจสอบการแจ้งเตือนเมื่อกรอกเลขบัตรประชาชนซ้ำในระบบ
*คำอธิบาย:* ตรวจสอบการตรวจสอบข้อมูลเลขประจำตัวประชาชนซ้ำซ้อนเพื่อป้องกันการบันทึกข้อมูลซ้ำ

* **Given** ผู้ใช้เข้าสู่หน้าจอสร้างประวัติผู้ป่วยใหม่ และมีเลขบัตรประชาชน "1234567890123" อยู่ในระบบแล้ว
* **When** ผู้ใช้ทำการป้อนเลขบัตรประชาชนเป็น "1234567890123" ซึ่งซ้ำกับในระบบ
* **And** กรอกข้อมูลจำเป็นอื่นๆ ครบถ้วนแล้วกดปุ่ม "บันทึกข้อมูล"
* **Then** ระบบจะต้องแสดงหน้าต่างแจ้งเตือนข้อผิดพลาดว่า "เลขประจำตัวประชาชนนี้มีอยู่ในระบบแล้ว"
* **And** ระบบจะต้องไม่อนุญาตให้สร้างรายการผู้ป่วยใหม่นี้

### 🚩 MR-005 (Functional): ตรวจสอบการทำงานของปุ่ม "อ่านบัตรประชาชน" (Smart Card)
*คำอธิบาย:* ทดสอบการปฏิสัมพันธ์กับอุปกรณ์อ่านบัตรสมาร์ทการ์ดเพื่อนำเข้าข้อมูลโดยอัตโนมัติ

* **Given** เครื่องคอมพิวเตอร์ทดสอบทำการเชื่อมต่ออุปกรณ์เครื่องอ่านบัตรประชาชน (Smart Card Reader) เรียบร้อยแล้ว
* **When** เสียบบัตรประจำตัวประชาชนจริงเข้าไปในช่องอ่านบัตร
* **And** กดปุ่ม "อ่านบัตรประชาชน" บนระบบหน้าจอเวชระเบียน
* **Then** ฟิลด์ข้อมูล ชื่อจริง นามสกุล วันเกิด และเลขบัตรประชาชน จะต้องถูกกรอกโดยอัตโนมัติอย่างถูกต้องแม่นยำตามข้อมูลในบัตร

### 🚩 MR-010 (Functional): การแก้ไขข้อมูลที่อยู่และเบอร์โทรศัพท์ติดต่อ (Edit Profile)
*คำอธิบาย:* ตรวจสอบว่าระบบสามารถบันทึกการอัปเดตข้อมูลรายละเอียดส่วนตัวที่อยู่และการติดต่อได้อย่างถูกต้อง

* **Given** ผู้ใช้ค้นหาข้อมูลผู้ป่วยพบและเข้ามาอยู่ในหน้าจอรายละเอียดข้อมูลผู้ป่วย (Patient Profile Detail Page)
* **When** คลิกปุ่ม "แก้ไขประวัติ" (Edit Profile)
* **And** ทำการเปลี่ยนค่าช่อง "เบอร์โทรศัพท์ติดต่อ" เป็น "099-999-9999" และแก้ไขรายละเอียด "ที่อยู่ปัจจุบัน"
* **And** คลิกปุ่ม "บันทึกข้อมูลการแก้ไข"
* **Then** ระบบจะแสดงข้อความอัปเดตข้อมูลสำเร็จ "แก้ไขข้อมูลเรียบร้อยแล้ว"
* **And** เมื่อกลับมาดูที่โปรไฟล์ผู้ป่วย ข้อมูลเบอร์โทรศัพท์และที่อยู่จะต้องเปลี่ยนเป็นค่าใหม่ทันที

### 🚩 MR-011 (Functional): แก้ไขผู้ติดต่อฉุกเฉิน (Emergency Contact)
*คำอธิบาย:* ตรวจสอบการทำงานส่วนการบันทึกข้อมูลผู้ติดต่อกรณีเกิดเหตุฉุกเฉินของผู้ป่วย

* **Given** ผู้ใช้อยู่ในหน้าแก้ไขข้อมูลประวัติผู้ป่วย (Patient Profile Edit Page)
* **When** ไปยังส่วนหัวข้อ "ผู้ติดต่อฉุกเฉิน" (Emergency Contact)
* **And** กรอกชื่อผู้ติดต่อฉุกเฉินเป็น "ContactName" และความสัมพันธ์เป็น "บิดา/มารดา" พร้อมเบอร์โทรศัพท์
* **And** คลิกปุ่ม "บันทึกข้อมูลการแก้ไข"
* **Then** รายละเอียดผู้ติดต่อฉุกเฉินจะต้องถูกเพิ่มเข้าประวัติผู้ป่วยอย่างถูกต้อง

### 🚩 MR-012 (UI/Functional): ตรวจสอบประวัติการแก้ไขข้อมูล (Audit Trail)
*คำอธิบาย:* ตรวจสอบการบันทึก Log การแก้ไขข้อมูลเพื่อการตรวจสอบย้อนหลัง

* **Given** ผู้ใช้เป็น Admin เข้ามาที่ประวัติส่วนตัวผู้ป่วยที่มีการแก้ไขข้อมูลมาก่อนหน้านี้
* **When** คลิกไปที่แท็บ "ประวัติการแก้ไขข้อมูล" (Audit Trail / Activity Log)
* **Then** ระบบจะต้องแสดงตารางประวัติระบุว่า "ผู้ใช้งานใด" ทำการแก้ไขข้อมูล "ฟิลด์ใด" "วันและเวลาใด" ได้อย่างละเอียดและถูกต้อง
